# The Vocabulary

*Reading time: ~7 min. Outcome: Fluency in the 10 terms that run Claude acquisitions.*

You do not need to become a technical user to run deals in Claude. But knowing these 10 terms will make you faster and prevent confusion when you read guides or talk to other operators.

## The 10 terms

| Term | What it means | Why it matters for deal work |
|---|---|---|
| **Skill** | A saved reusable prompt that runs multi-step tasks end to end. | You can build a skill once (e.g. "qualify this target against my AQ criteria") and re-run it on every new business with one command. |
| **Prompt** | The instruction you give Claude. Can be a question, a command, or a full task spec. | Better prompts get better results. "Summarize this" is weak. "Extract the red flags from this CIM and score them against my buy box" is owner-level. |
| **Loop** | When Claude runs a task that takes multiple steps without you intervening. | Long agentic loops let you say "build a target list from this vertical" and walk away while Claude sources, screens, and writes the report. |
| **Agent** | Claude acting autonomously to complete a task using tools and reasoning. | In agentic mode, Claude decides what steps to take, what to search, and how to structure the output — you give the goal, not the procedure. |
| **Sub-agent** | A separate Claude instance spun up inside a task to handle part of the job. | When you run a deep research task, Claude may launch sub-agents to pull comps, analyze financials, and draft the memo in parallel. You see one output; Claude managed the team. |
| **Project** | A persistent workspace where uploaded files and custom instructions are always available. | Your acquisition Project holds your buy box and method. Every conversation inside it starts with that context loaded. |
| **Context window** | The total amount of text Claude can "see" at once (conversation + files + instructions). | Claude Pro has a large context window, so you can upload a 100-page CIM and ask detailed questions without summarizing first. |
| **Compaction** | What happens when a conversation gets too long: Claude compresses the middle to stay under the context limit. | Files survive compaction. Chat memory does not. Save important outputs as files or into your Project. |
| **Connector** | A data source Claude can pull from (web search, file uploads, eventually APIs). | Web search is the connector that powers on-market listing monitoring and comp research. |
| **Artifact** | A formatted output Claude generates in a side panel (tables, documents, code). | When Claude builds you a target scorecard or valuation model, it appears as an artifact you can copy, download, or iterate on. |

## How these fit together in a real acquisition task

You open your **Project** (persistent context). You upload a business listing PDF (now in the **context window**). You run a saved **skill** with a good **prompt**: "Qualify this business against my AQ criteria and flag any reds." Claude goes **agentic**, pulls comps via the web **connector**, maybe spins up a **sub-agent** to research the industry. The task runs as a **loop** (multiple steps, no hand-holding). Claude delivers a scorecard as an **artifact**. The conversation gets long, so older messages **compact**, but your uploaded listing and Project files stay available.

That is the system.

## What good looks like when you know the vocabulary

- You can describe what you want Claude to do without guessing at features.
- You know the difference between "ask Claude a question" and "run an agentic loop."
- You understand why saving outputs as files matters (compaction).
- You can read other users' workflows and adapt them to deal work.
